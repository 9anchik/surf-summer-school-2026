# User stories

> Этап 2. Пользовательские истории. Формат: «Как <роль>, я хочу <действие>, чтобы <ценность>».
> Каждой истории — критерии приёмки.
>
> **Скоуп: клиентское приложение и API для него** (роль «Клиент»). Функции маршала и
> владельца обеспечиваются существующей инфраструктурой и здесь не описываются.

| ID | Как (роль) | Хочу (действие) | Чтобы (ценность) | Критерии приёмки | FR |
| :-- | :-- | :-- | :-- | :-- | :-- |
| US-1 | Клиент | зарегистрироваться по имени и номеру телефона без пароля | быстро начать пользоваться без барьеров | Указав имя и телефон, получаю SMS с одноразовым кодом (OTP); ввожу код — получаю доступ; повторный вход — по номеру телефона; сложный пароль не требуется. | FR-1, FR-2, FR-43 |
| US-2 | Клиент | видеть список предстоящих слотов заездов | выбрать подходящий заезд | По умолчанию показываются ВСЕ предстоящие слоты (`only_available=false`) на ближайшие 7 дней, отсортированные по времени старта; слот показывает дату/время старта, время прибытия (старт − 15 мин), конфигурацию трассы, маршала, цену, всего/свободно мест; заполненные (свободно = 0) и отменённые центром слоты остаются в списке с пометкой «Мест нет» и неактивной CTA записи; при отсутствии слотов — пустое состояние «Заездов пока нет». | FR-9 |
| US-3 | Клиент | фильтровать список слотов | быстро находить подходящий вариант | Могу фильтровать по дате/периоду (с пресетами «Сегодня / Эта неделя / Выходные»), конфигурации трассы (короткая/длинная), наличию свободных мест, маршалу; внутри группы (конфигурация, маршал — множественный выбор) значения по «ИЛИ», между группами — по «И»; границы дат включительные, в локальной зоне центра; дефолт — без фильтров; пустой справочник маршалов — фильтр скрыт; активные фильтры отображаются; список обновляется по выбору. | FR-38 |
| US-4 | Клиент | открыть карточку слота со всеми параметрами | понять детали перед записью | Карточка показывает дату/время старта, время прибытия (старт − 15 мин), конфигурацию трассы и её описание, маршала, цену, всего/свободно мест, доступность прокатной экипировки, адрес центра; место сбора отображается на карте (Яндекс.Карты) с точкой `meeting_point`; при недоступности карты — fallback с текстовым адресом и координатами. | FR-9a, FR-44 |
| US-5 | Клиент | записаться на выбранный слот | забронировать место на заезд | После подтверждения создаётся запись; свободные места уменьшаются; запись появляется в «Моих бронированиях»; при создании передаётся `Idempotency-Key` для защиты от дублей. | FR-10 |
| US-6 | Клиент | при записи указать свою или прокатную экипировку | приехать со своим шлемом или взять напрокат | Можно выбрать «своя» / «напрокат» для каждого места; «напрокат» уменьшает прокатный фонд экипировки, «своя» — нет. | FR-11, FR-14 |
| US-7 | Клиент | одной записью забронировать несколько мест (себя + гости) | записать компанию за раз | Можно указать 1–3 места в одной записи при наличии свободных; для каждого гостя выбирается вариант экипировки; имена/контакты гостей не собираются. | FR-12 |
| US-8 | Клиент | быть защищённым от записи сверх лимита | не приехать на занятое/несуществующее место | Лимит мест `max_seats = min(free_seats, track_config.capacity_cap, 3)` (свободные места, потолок конфигурации трассы — короткая ≤8, длинная ≤14, лимит 3 на бронь); прокат проверяется **отдельно**: `rental_count ≤ free_rental_equipment`; при гонке запросов двойная бронь/овербукинг исключены. | FR-13, FR-15 |
| US-9 | Клиент | видеть список своих бронирований | контролировать предстоящие и прошедшие заезды | Вижу свои записи (предстоящие и прошедшие) со статусом, параметрами слота, числом мест и вариантом экипировки; история пагинируется; отменённые и поздние отмены также отображаются. | FR-35a |
| US-10 | Клиент | отменить свою запись до старта | освободить место, если планы изменились | Запись отменяется только целиком (частичная отмена гостя не поддерживается); при ранней отмене (≥3 ч, включая ровно 3 ч) освобождаются ВСЕ места и прокатные комплекты экипировки брони; при поздней (<3 ч) запись помечается «поздняя отмена», места не освобождаются, штрафов нет; время в UTC отображается в локальной зоне, статус отмены определяет сервер. | FR-16, FR-17, FR-18 |
| US-11 | Клиент | видеть, что заезд отменён центром | понимать причину и не приезжать зря | Если центр отменил заезд (например, из-за погоды — сильный дождь, гололёд), бронь не удаляется, а переходит в статус «Отменён центром» с указанием причины; приходит push-уведомление; повторная запись на этот слот запрещена (кнопка неактивна). | FR-46 |
| US-12 | Клиент | видеть цену заезда | понимать стоимость заранее | В карточке и при подтверждении показана итоговая цена (`price_total`); оплата офлайн (наличные/перевод на карту), запись фиксируется без онлайн-оплаты. | FR-30, FR-45 |
| US-13 | Клиент | получать напоминание о предстоящей записи | не забыть и не пропустить заезд | Заблаговременно до старта (за 24 ч и за 3 ч) приходит напоминание через **системный push** (нативное/гибридное приложение, APNs/FCM); SMS/email — Phase 2. *(Should)* | FR-33 |
| US-14 | Клиент | просматривать ранее загруженные данные офлайн | ориентироваться у трассы при нестабильной связи | При отсутствии сети вижу ранее загруженный список слотов и свои брони с пометкой «данные могли устареть»; изменяющие действия (бронь, отмена) офлайн недоступны с понятным сообщением; единый Error/Retry для повторной загрузки. | FR-9, FR-35a |
| US-15 | Клиент | быть защищённым от двойной брони при сбое сети | не получить дубль и не переплатить | Клиент при создании брони передаёт обязательный `Idempotency-Key`; при неопределённом результате повтор с тем же ключом и телом даёт идентичный ответ и не создаёт дубль. | FR-10, FR-13 |
| US-16 | Клиент | видеть понятные сообщения об ошибках | понять причину и что делать дальше | Для ключевых операций ошибки отображаются по единой матрице (код → реакция): нехватка мест/экипировки, двойная бронь, отменённый/стартовавший слот, неверный код, лимит запросов, серверная ошибка; текст локализован; где уместно — предлагается повтор. | FR-10, FR-16 |
| US-17 | Клиент | управлять своим профилем | актуализировать данные или удалить аккаунт | Могу просматривать и редактировать имя и номер телефона; могу выйти из аккаунта; могу удалить аккаунт — при этом активные брони аннулируются (освобождают места и экипировку), а прошедшие брони анонимизируются. | FR-47, FR-48, FR-49 |

> **Вне скоупа.** Истории маршала и владельца, а также оценки маршалам обеспечиваются
> существующей инфраструктурой и в клиентском приложении не реализуются.
