# Карта маршрута

**ID:** LOGIC-07  
**Тип:** Логика  
**Домен:** 09. Логики  
**Приоритет:** Low  
**Статус:** На согласовании  
**Функциональные блоки:** FB-MAP-001

---

## Обзор

Логика построения маршрута от текущего местоположения до адреса ресурса: запрос маршрута, отображение полигона/полилинии, расчёт времени в пути.

### User Story

> Как пользователь, я хочу видеть маршрут до точки бронирования, чтобы быстро добраться вовремя.

---

## Флоу

```mermaid
flowchart TD
  Start[Пользователь нажимает "Построить маршрут"] --> GetLocation[Получить текущее местоположение]
  GetLocation --> RoutingAPI[GET /routing?from=&to=&mode=driving]
  RoutingAPI --> Map[Нарисовать polyline и показать ETA]
```

---

## Подход

- Использовать внешний routing provider (OSRM / Google Directions / MAPBOX).  
- Клиент запрашивает маршрут с параметром mode (driving/walking/bicycling).  
- Отображать линию маршрута, маркеры и ETA.  
- Предусмотреть кеширование последнего маршрута и offline-fallback (напр., открытие в стороннем навигаторе).

---

## API запросы

### GET /routing?from=lat,lng&to=lat,lng&mode=

**Обработка ответа:**

| Код | Действие |
|-----|----------|
| 200 | Отрисовать polyline + показать время/расстояние |
| 4xx/5xx | Показать "Не удалось построить маршрут" и предложить открыть в Google Maps/Apple Maps |

---

## Критерии приёмки

| ID | Критерий |
|----|---------|
| AC-001 | Маршрут отображается с корректным полилинием и ETA |
| AC-002 | При отсутствии сети пользователь может открыть маршрут в стороннем приложении |

---

## Обработка ошибок

| Тип ошибки | Контекст | Действие |
|------------|----------|----------|
| Нет доступа к геолокации | Получение местоположения | Попросить разрешение или предложить ввести адрес вручную |
| Routing provider failure | Запрос маршрута | Показать fallback-ссылку на сторонние карты |
