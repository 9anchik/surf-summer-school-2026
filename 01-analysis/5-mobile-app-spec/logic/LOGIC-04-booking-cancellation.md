# Отмена брони

**ID:** LOGIC-04  
**Тип:** Логика  
**Домен:** 09. Логики  
**Приоритет:** High  
**Статус:** Актуален  
**Функциональные блоки:** FB-BOOK-002

---

## Обзор

Логика отмены брони с проверкой условий возврата, ограничения по времени и пробуждением слота.

### User Story

> Как пользователь, я хочу отменить бронь и при необходимости получить возврат средств.

---

## Флоу

```mermaid
flowchart TD
  CancelTap --> CheckPolicy[Проверить политику отмены]
  CheckPolicy -->|Allowed| CancelAPI[POST /bookings/{id}/cancel]
  CancelAPI -->|OK| ReleaseSlot[Освободить слот и уведомить]
  CheckPolicy -->|NotAllowed| Deny[Показать причину отказа]
```

---

## Правила

- Разрешена отмена до N минут до начала (config per resource).
- Для частичной/полной оплаты учитывать политику возврата.
- При отмене: отмечать booking.status = cancelled, увеличивать доступность слота, отправлять уведомления.
- Логирование причин отмены для аналитики.

---

## API запросы

### POST /bookings/{id}/cancel

**Body:** { "reason": "user_request" }

**Обработка ответа:**

| Код | Действие |
|-----|----------|
| 200 | Бронь отменена, вернуть refund_status |
| 403 | Запрещено (слишком поздно) |
| 404 | Бронь не найдена |

---

## Критерии приёмки

| ID | Критерий |
|----|---------|
| AC-001 | Пользователь может отменить бронь в allowed window и получить статус cancelled |
| AC-002 | Слот снова доступен для бронирования после отмены |

---

## Обработка ошибок

| Тип ошибки | Контекст | Действие |
|------------|----------|----------|
| Отмена запрещена | Политика отмены | Показать четкую причину и инструкции |
| Ошибка платежа при возврате | Refund pipeline | Сохранить задачу на повторный возврат и оповестить поддержку |
