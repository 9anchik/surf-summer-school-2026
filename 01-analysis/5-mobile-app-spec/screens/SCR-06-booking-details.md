# Детали брони и Отмена

**ID:** SCR-06  
**Тип:** Экран  
**Домен:** Mobile App / Booking details  
**Приоритет:** High  
**Статус:** Актуален  
**Функциональные блоки:** FB-BOOKING-003

---

## Обзор

Детали конкретной брони с таймером до старта, политикой отмены (ранняя / поздняя) и CTA "Отменить". Политика и поведение определяются сервером (FR-16..FR-18).

---

## Навигация

Вход: SCR-05 (мои брони) / push deep link.  
Исход: После успешной отмены — возврат в SCR-05.

---

## Входные данные

booking_id — GET /bookings/{id} возвращает slot, timer, policy, price, status.

---

## Применяемые логики

LOGIC-04 (отмена брони) — проверка allowed window и вызов POST /bookings/{id}/cancel.

---

## Используемые запросы

### POST /bookings/{id}/cancel

**Body:** { reason }

**Обработка:** 200 — success (refund_status), 403 — forbidden (too late), 404 — not found.

---

## Состояния

Показывать динамический текст политики отмены; при таймере <3ч показывать предупреждение для поздней отмены.

---

## Связанные требования

### Функциональные (REQ-FUNC-*)

| ID | Название | Приоритет |
|---|---|---|
| FR-16 | Возможность отмены брони | Must |
| FR-17 | Ранняя отмена (>=3ч) — освобождение мест | Must |
| FR-18 | Поздняя отмена (<3ч) — late_cancel | Must |
| FR-46 | Отмена центром — отображение причины | Must |

### Интеграции (REQ-INT-*)

| ID | Название | API operationId (контракт) |
|---|---|---|
| REQ-INT-BOOKINGS | Backend Bookings API | getBooking — GET /bookings/{bookingId}; cancelBooking — POST /bookings/{bookingId}/cancel — см. /01-analysis/api/bookings/bookings.yaml


## Критерии приёмки

- Модалка подтверждения различает раннюю и позднюю отмену
- При успешной отмене слот освобождается (серверная логика) и отображается в списке
