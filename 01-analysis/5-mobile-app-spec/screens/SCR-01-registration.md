## Обзор

Экран ввода номера и экран ввода OTP. Быстрая регистрация/вход по SMS-OTP (FR-1, FR-2, FR-43). Показывается при первом запуске или истечении сессии.

### User Story

> Как новый или вернувшийся клиент, я хочу подтвердить номер через SMS, чтобы быстро попасть в приложение.

### Бизнес-ценность

- Уменьшение барьера входа, рост конверсии в активных пользователей.

## Используемые запросы

### POST /auth/otp/send

**Триггер:** Тап "Получить код"  
**Body:** { "phone": "+7..." }

**Обработка ответа:** см. LOGIC-01

### POST /auth/otp/verify

**Триггер:** Ввод кода  
**Body:** { "phone": "+7...", "code": "1234" }

## Критерии приёмки

| ID | Критерий |
|---|---|
| AC-001 | При вводе валидного номера приходит SMS и пользователь входит |
| AC-002 | Неверный код показывает понятную ошибку |

---

## Обработка ошибок

| Тип ошибки | Контекст | Действие |
|---|---|---|
| Rate limit | /auth/otp/send | Показ таймера и блокировки |
| Неверный код | /auth/otp/verify | Подсказка, счетчик попыток |
