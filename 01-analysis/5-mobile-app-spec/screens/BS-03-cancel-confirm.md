# Подтверждение отмены (Bottom Sheet)

**ID:** BS-03  
**Тип:** Bottom Sheet / Modal  
**Домен:** Mobile App / Booking cancellation  
**Приоритет:** High  
**Статус:** Актуален  
**Функциональные блоки:** FB-BOOK-002

---

## Обзор

Модалка подтверждения отмены брони. Два варианта текста: ранняя отмена (≥3ч) и поздняя отмена (<3ч). Подтверждение вызывает POST /bookings/{id}/cancel.

---

## Логика

LOGIC-04 — проверка политики отмены, отображение разного текста и кнопок в зависимости от таймера.

---

## Связанные требования

### Функциональные (REQ-FUNC-*)

| ID | Название | Приоритет |
|---|---|---|
| FR-16 | Возможность отмены брони | Must |
| FR-17 | Ранняя отмена (>=3ч) | Must |
| FR-18 | Поздняя отмена (<3ч) | Must |

### Интеграции (REQ-INT-*)

| ID | Название | API operationId (контракт) |
|---|---|---|
| REQ-INT-BOOKINGS | Backend Bookings API | cancelBooking — POST /bookings/{bookingId}/cancel — см. /01-analysis/api/bookings/bookings.yaml


## Критерии приёмки

- Текст различается для ранней/поздней отмены
- При подтверждении отправляется POST и UI обновляется
