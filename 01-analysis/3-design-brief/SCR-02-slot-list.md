# SCR-02: Список заездов (Главная) — Постановка на дизайн

## 1. Метаданные

| Поле | Значение |
| :-- | :-- |
| **ID экрана** | SCR-02 |
| **Название** | Список заездов (Главная) |
| **Роль пользователя** | Клиент |
| **Тип экрана** | Корневой (Bottom Navigation → Главная) |
| **Связанные US** | US-2, US-3 |
| **Связанные UC** | UC-3 |
| **Ключевые FR** | FR-9, FR-38 |
| **Связанные NFR** | NFR-1, NFR-6, NFR-21, NFR-24 |

## 2. Цель экрана и JTBD

- **JTBD:** Когда я хочу покататься, я хочу быстро просмотреть, когда и на какую трассу есть места, чтобы выбрать удобное время и записаться.
- **Бизнес-цель:** Максимально быстро довести пользователя до карточки слота и бронирования → конверсия в запись.
- **Главный вопрос:** «Есть ли места на удобное мне время?»

##  3. Пользователь и контекст использования

- **Окружение:** На улице, на ярком солнце, возможно в движении.
- **Эмоция:** Предвкушение, быстрый выбор.
- **Устройство:** Mobile-first, одной рукой.
- **Время на экране:** 3–10 секунд на сканирование списка.

## 4. Информационная архитектура и контент

### 4.1. Иерархия информации (в карточке слота)
1. **Дата и время старта** (крупно, акцент).
2. **Конфигурация трассы** (бейдж: «Короткая / Новички» или «Длинная / Опытные»).
3. **Статус мест** («Свободно 5 из 8» / «Мест нет»).
4. **Маршал** (имя, опц. фото).
5. **Цена** (вторично).

### 4.2. Поля и данные

| Поле | Обязательность | Формат / Лимиты | Примечание |
| :-- | :-- | :-- | :-- |
| Дата | Обязательно | `Сб, 15 июня` | Локальная зона центра |
| Время старта | Обязательно | `18:00` | Крупно |
| Конфигурация трассы | Обязательно | Бейдж с цветом | Короткая — синий, Длинная — оранжевый |
| Маршал | Обязательно | Имя | Опц. маленькое фото |
| Свободно мест | Обязательно | `5 / 8` | Цветовая индикация |
| Цена | Обязательно | `2 500 ₽` | Вторичный цвет |
| Статус слота | Обязательно | Активен / Мест нет / Отменён | Влияет на CTA |

### 4.3. Ограничения контента
- Имя маршала — до 20 символов (иначе сокращать с `...`).
- Дефолтный горизонт — 7 дней (R-027).

## 5. Пользовательские сценарии

### 5.1. Основные потоки
1. Открывается список слотов на 7 дней.
2. Пользователь скроллит, сканирует карточки.
3. Тапает на карточку → раскрытие карточки слота (SCR-03) как Bottom Sheet.

### 5.2. Альтернативные потоки
- **A1. Применение фильтров:** Открывает Bottom Sheet с фильтрами → выбирает → список пересобирается.
- **A2. Быстрые пресеты дат:** Тапает «Сегодня / Выходные / Неделя» → список обновляется.
- **A3. Pull-to-refresh:** Потянул вниз → обновление списка.
- **A4. Сброс фильтров:** Кнопка «Сбросить» → возврат к дефолту (7 дней, без фильтров).

### 5.3. Точки принятия решений
- Выбор карточки — единственный CTA на элементе списка.
- Фильтры — опциональны, не блокируют основной сценарий.

## 6. Состояния экрана и компонентов

### 6.1. Загрузка
- **Скелетон карточек** (шиммер) при первой загрузке.
- **Pull-to-refresh** — кастомный спиннер (колесо картинга).

### 6.2. Пустые состояния
- **Нет слотов на 7 дней:** Иллюстрация «Трасса готовится», текст «На ближайшие дни заездов нет», CTA «Посмотреть другие даты».
- **Фильтр не дал результатов:** «Ничего не найдено. Попробуйте изменить фильтры», CTA «Сбросить фильтры».

### 6.3. Ошибки
- **Ошибка загрузки (5xx):** Иллюстрация «Что-то пошло не так», кнопка «Попробовать снова».
- **Карточка слота недоступна:** Состояние «Мест нет» / «Отменён центром» — полупрозрачная карточка, CTA неактивна.

### 6.4. Офлайн-режим
- Баннер «Нет подключения. Данные могут быть устаревшими».
- Показывается кэш списка (read-only).
- Тап по карточке → SCR-03 (из кэша).

### 6.5. Ограничения
- Заполненные и отменённые слоты НЕ скрываются (только если не включён фильтр «Только свободные»).
- Сортировка — по времени старта (возрастание).

## 7. Навигация и переходы

- **Вход на экран:** Тап по иконке «Главная» в Bottom Navigation / первый вход после авторизации.
- **Выход:** Тап по карточке → SCR-03 (Bottom Sheet) / Тап по другой иконке Bottom Nav.
- **Поведение Back:** На этом экране системный Back = выход из приложения (с подтверждением, опц.).
- **Deep Links:** Не применимо (корневой экран).

## 8. Визуальный язык и настроение

- **Вайб:** Динамика, уличная культура, сканируемый список.
- **Цветовая семантика:**
  - 🔵 Короткая трасса / новички.
  - 🟠 Длинная трасса / опытные.
  - 🟢 Много мест.
  - 🟡 Мало мест (≤2).
  - 🔴 Мест нет / отменён.
- **Типографика:** Время старта — крупно (H2), дата — средне, остальное — body.
- **Карточки:** Скруглённые углы, лёгкая тень, разделитель — пространство.

## 9. Технические ограничения

- **Тач-зоны:** Вся карточка — кликабельная, ≥44pt по высоте.
- **Safe Area:** Bottom Navigation в безопасной зоне.
- **Контраст:** WCAG AA (чтение на солнце).
- **Системные шрифты:** Dynamic Type — карточка должна растягиваться по высоте.

## 10. Анимации и микроинтеракции

- **Появление карточек:** Лёгкий slide-up с каскадной задержкой (при первой загрузке).
- **Pull-to-refresh:** Кастомная анимация (колесо картинга).
- **Тап по карточке:** Лёгкое затемнение (press state).
- **Применение фильтра:** Плавное пересобираение списка (fade + reposition).

## 11. Критерии приёмки

- [ ] Карточка слота читается за 1 секунду.
- [ ] Проработаны состояния: загрузка, пустота (нет слотов / фильтр не дал), ошибка, офлайн.
- [ ] Учтены состояния «Мест нет» и «Отменён центром».
- [ ] Фильтры не перекрывают основной контент (Bottom Sheet).
- [ ] Pull-to-refresh работает.
- [ ] Тач-зоны и контраст соблюдены.

## 12. Артефакты на выходе

1. Экран списка: все состояния (загрузка, пустота, ошибка, офлайн).
2. Компонент «Карточка слота»: 4 состояния (активен, мало мест, мест нет, отменён).
3. Bottom Sheet фильтров: все элементы + пресеты дат.
4. Прототип: скролл, тап по карточке, применение фильтра, pull-to-refresh.
5. Спецификация: отступы, цвета статусов, тексты empty states.
6. Ассеты: иллюстрации для empty states (SVG), иконка pull-to-refresh.
