## 2. Цель экрана и JTBD

- **JTBD:** Когда я решил записаться, я хочу указать, сколько нас будет и кому нужна экипировка, чтобы всё было готово к моему приезду.
- **Бизнес-цель:** Собрать данные для распределения инвентаря и мест, предотвратить овербукинг, конвертировать просмотр в бронь.

## 3. Пользователь и контекст использования

- **Окружение:** Фокус, внимание к деталям.
- **Эмоция:** Ответственность (за себя и гостей).
- **Устройство:** Mobile-first.
- **Время на экране:** 20–40 секунд.

## 4. Информационная архитектура и контент

### 4.1. Иерархия информации
1. **Степпер количества мест** (1–3).
2. **Список участников** (для каждого — выбор экипировки).
3. **Итоговая цена** (динамическая).
4. **CTA** «Подтвердить и забронировать».
5. **Экран успеха** (после подтверждения).

### 4.2. Поля и данные

| Поле | Обязательность | Формат / Лимиты | Примечание |
| :-- | :-- | :-- | :-- |
| Количество мест | Обязательно | Степпер 1–3 | Лимит FR-12 |
| Для каждого места: экипировка | Обязательно | Тоггл «Своя / Прокат» | Влияет на цену |
| Итоговая цена | Обязательно | `5 600 ₽` | Динамический пересчёт |
| Свободно мест / проката | Обязательно | `Осталось: 3 места, 2 комплекта` | Подсказка |

### 4.3. Ограничения контента
- Максимум 3 места на бронь (FR-12).
- Лимит мест = `min(free_seats, track_config.capacity_cap, 3)` (FR-13).
- Лимит проката = `free_rental_equipment` (FR-14).

## 5. Пользовательские сценарии

### 5.1. Основные потоки
1. Открывается модальное окно (slide-up).
2. Пользователь выбирает количество мест (например, 2).
3. Появляются два блока с тогглами экипировки.
4. Видит итоговую цену.
5. Тапает «Подтвердить» → загрузка → экран успеха (внутри модалки).
6. На экране успеха: резюме брони, время прибытия, CTA «Добавить в календарь» / «Вернуться на главную».

### 5.2. Альтернативные потоки
- **A1. Изменение количества мест:** Увеличил/уменьшил → блоки экипировки пересобираются.
- **A2. Переключение экипировки:** Тап по тогглу → цена пересчитывается мгновенно.
- **A3. Закрыть без бронирования:** Swipe down / тап по фону → модалка закрывается.
- **A4. После успеха:** Тап «Вернуться на главную» → закрытие модалки и возврат к SCR-02.

### 5.3. Точки принятия решений
- **Выбор экипировки для каждого места** — ключевой момент.
- **Итоговая цена** — должна обновляться мгновенно.

## 6. Состояния экрана и компонентов

### 6.1. Загрузка
- Спиннер внутри CTA при отправке запроса.

### 6.2. Пустые состояния
- Не применимо.

### 6.3. Ошибки
- **Не хватает проката:** Если выбрано 2 проката, а свободно 1 → тоггл блокируется, подсказка «Свободен только 1 комплект, выберите «Своя экипировка» для второго».
- **Гонка запросов (слот заполнился):** При тапе «Подтвердить» → Bottom Sheet «Места только что закончились. Выберите другое время», кнопка «К списку заездов».
- **Двойная бронь (double_booking):** «Вы уже записаны на этот заезд. Перейти к брони?» → кнопка «К моим заездам».
- **Сетевой сбой:** «Не удалось забронировать. Проверьте сеть», кнопка «Повторить» (с тем же Idempotency-Key).

### 6.4. Офлайн-режим
- CTA неактивна, баннер «Нет подключения».

### 6.5. Ограничения
- Степпер мест ограничен доступным количеством (`max_seats`).
- Тогглы проката ограничены `free_rental_equipment`.

## 7. Навигация и переходы

- **Вход на экран:** Тап по CTA «Записаться» в SCR-03 (slide-up модалка).
- **Выход:** Swipe down / тап по фону / системный Back → возврат в SCR-03.
- **Поведение Back:** Закрытие модалки без бронирования.
- **После успеха:** Модалка показывает экран успеха → тап по CTA → закрытие модалки и возврат к SCR-02.

## 8. Визуальный язык и настроение

- **Вайб:** Контроль, чистота, мгновенная обратная связь.
- **Цветовая семантика:**
  - 🟢 Своя экипировка.
  - 🔵 Прокат.
  - 🔴 Недоступно (заблокированный тоггл).
  - 🟢 Успех — зелёная галочка.
- **Типографика:** Итоговая цена — крупно, акцент. Время прибытия — янтарный акцент.
- **Тогглы:** Крупные, с иконками (шлем).

## 9. Технические ограничения

- **Тач-зоны:** Тогглы и CTA — ≥44pt.
- **Клавиатуры:** Не нужны (только степпер и тогглы).
- **Safe Area:** CTA не перекрывается Home Indicator.
- **Контраст:** WCAG AA.
- **Системный календарь:** Интеграция через системные API (EventKit / Intent).

## 10. Анимации и микроинтеракции

- **Появление модалки:** Slide-up с spring-анимацией.
- **Изменение количества мест:** Плавное появление/исчезновение блоков экипировки.
- **Пересчёт цены:** Лёгкий bounce цифры.
- **CTA:** Press state, при загрузке — спиннер.
- **Экран успеха:** Lottie-анимация галочки (check).

## 11. Критерии приёмки

- [ ] Лимит 3 места соблюдён и понятен.
- [ ] Итоговая цена обновляется мгновенно.
- [ ] Учтена нехватка прокатной экипировки (блокировка тоггла + подсказка).
- [ ] Проработаны ошибки: гонка запросов, двойная бронь, сеть.
- [ ] Экран успеха с резюме брони и временем прибытия.
- [ ] Возможность добавить в календарь.
- [ ] Тач-зоны и контраст соблюдены.

## 12. Артефакты на выходе

1. Модальное окно: все состояния (успех, нехватка проката, гонка запросов, офлайн).
2. Экран успеха: резюме брони, время прибытия, CTA.
3. Компоненты: степпер мест, тоггл экипировки (3 состояния), блок участника.
4. Lottie-анимация галочки (или запрос к моушн-дизайнеру).
5. Прототип: выбор мест, переключение экипировки, подтверждение, экран успеха, ошибки.
6. Спецификация: отступы, тексты ошибок, логика пересчёта.
7. Ассеты: иконки шлема (своя / прокат).
