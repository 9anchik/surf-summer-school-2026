# SCR-03: Карточка слота — Постановка на дизайн

## 1. Метаданные

| Поле | Значение |
| :-- | :-- |
| **ID экрана** | SCR-03 |
| **Название** | Карточка слота |
| **Роль пользователя** | Клиент |
| **Тип экрана** | Детализация / Bottom Sheet (раскрывается поверх списка) |
| **Связанные US** | US-4 |
| **Связанные UC** | UC-5 |
| **Ключевые FR** | FR-9a, FR-44 |
| **Связанные NFR** | NFR-23, NFR-26 |

## 2. Цель экрана и JTBD

- **JTBD:** Когда я выбрал время в списке, я хочу быстро увидеть все детали заезда (трассу, маршала, место сбора), чтобы принять решение о записи, не уходя с главного экрана.
- **Бизнес-цель:** Дать исчерпывающую информацию, снять тревожность («где быть? что брать?»), подтолкнуть к кнопке «Записаться».
- **Главный вопрос:** «Мне это подходит? Где собираться?»

## 3. Пользователь и контекст использования

- **Окружение:** На улице, на ярком солнце, возможно в движении.
- **Эмоция:** Вдумчивый выбор, предвкушение.
- **Устройство:** Mobile-first, одной рукой.
- **Время на экране:** 15–30 секунд (быстрый просмотр деталей).

## 4. Информационная архитектура и контент

### 4.1. Иерархия информации
1. **Время старта + Время прибытия** (критично!).
2. **Конфигурация трассы** + краткое описание.
3. **Маршал** (имя, фото опц.).
4. **Место сбора** (адрес + мини-карта).
5. **Цена** (заезд + прокат).
6. **CTA** «Записаться» (sticky внизу карточки).

### 4.2. Поля и данные

| Поле | Обязательность | Формат / Лимиты | Примечание |
| :-- | :-- | :-- | :-- |
| Дата и время старта | Обязательно | `Сб, 15 июня, 18:00` | Крупно |
| **Время прибытия** | Обязательно | `Приезжайте к 17:45` (старт − 15 мин) | **Акцентировать!** |
| Конфигурация трассы | Обязательно | Бейдж + описание (до 150 симв.) | Короткая / Длинная |
| Маршал | Обязательно | Имя, фото (опц.) | — |
| Цена заезда | Обязательно | `2 500 ₽` | — |
| Цена проката | Обязательно | `+ 300 ₽` | Вторично |
| Адрес центра | Обязательно | Текстом | — |
| Точка сбора (meeting_point) | Обязательно | Координаты + мини-карта | Я.Карты |
| Свободно мест | Обязательно | `5 / 8` | — |
| Доступно проката | Обязательно | `3 комплекта` | — |

### 4.3. Ограничения контента
- Описание трассы — до 150 символов (иначе `...` + «Подробнее»).
- Мини-карта — высота ~120pt, интерактивная (зум/скролл отключены, только тап для открытия полной карты).

## 5. Пользовательские сценарии

### 5.1. Основные потоки
1. Пользователь тапает по карточке в списке (SCR-02) → карточка слота раскрывается как Bottom Sheet (slide-up).
2. Изучает время, трассу, маршала.
3. Смотрит мини-карту места сбора.
4. Тапает «Записаться» → открывается SCR-04 (модальное окно оформления брони).

### 5.2. Альтернативные потоки
- **A1. Развернуть описание трассы:** Тап по «Подробнее» → раскрытие текста (expandable).
- **A2. Открыть карту в Яндекс.Навигаторе:** Тап по мини-карте → deep link в Яндекс.Карты.
- **A3. Закрыть карточку:** Swipe down / тап по фону → возврат к списку.
- **A4. Поделиться слотом:** (Won't в MVP, иконка неактивна).

### 5.3. Точки принятия решений
- **Время прибытия** — ключевой акцент, чтобы клиент не опоздал.
- **CTA «Записаться»** — всегда видна (sticky внизу карточки).

## 6. Состояния экрана и компонентов

### 6.1. Загрузка
- Скелетон карточки (шиммер) при открытии.

### 6.2. Пустые состояния
- Не применимо (карточка всегда открывается с данными слота).

### 6.3. Ошибки
- **Слот исчез (отменён между списком и карточкой):** Алерт «Заезд отменён: <причина>», кнопка «Закрыть».
- **Карта не загрузилась:** Fallback — текстовый адрес + кнопка «Открыть в Яндекс.Картах».
- **Ошибка загрузки деталей:** «Не удалось загрузить», кнопка «Попробовать снова».

### 6.4. Офлайн-режим
- Показывается кэш карточки с пометкой «Данные могут быть устаревшими».
- CTA «Записаться» неактивна.

### 6.5. Ограничения
- **Мест нет:** CTA меняется на «Мест нет» (неактивна, серый).
- **Отменён центром:** CTA скрыта, вверху алерт с причиной.
- **Заезд уже стартовал:** CTA «Заезд начался» (неактивна).

## 7. Навигация и переходы

- **Вход на экран:** Тап по карточке в SCR-002 (списке) → Bottom Sheet slide-up.
- **Выход:** Swipe down / тап по фону / системный Back → возврат к списку.
- **Поведение Back:** Возврат в список с сохранением фильтров и позиции скролла.
- **Из карточки:** CTA «Записаться» → SCR-004 (модальное окно, slide-up поверх карточки).
- **Deep Links:** Тап по пушу «Через 3 часа заезд» → открывается SCR-002 (список) с автоматическим раскрытием карточки нужного слота.

## 8. Визуальный язык и настроение

- **Вайб:** Инструктаж, безопасность, информативность.
- **Цветовая семантика:**
  - Время прибытия — янтарный акцент (внимание!).
  - 🔵/🟠 Конфигурация трассы — как в списке.
- **Типографика:** Время старта — H2, время прибытия — выделено цветом и иконкой.
- **Мини-карта:** Скруглённые углы, кастомная метка (логотип центра).

## 9. Технические ограничения

- **Тач-зоны:** CTA «Записаться» — ≥44pt, sticky внизу карточки.
- **Safe Area:** CTA не перекрывается Home Indicator.
- **Карта:** Интеграция с Яндекс.Картами (NFR-26), fallback при недоступности.
- **Контраст:** WCAG AA (чтение на солнце).
- **Высота Bottom Sheet:** ~70% экрана (остальное — затемнённый фон списка).

## 10. Анимации и микроинтеракции

- **Появление карточки:** Slide-up с spring-анимацией (из карточки списка в Bottom Sheet).
- **Появление карты:** Плавный fade-in.
- **CTA:** Press state, при загрузке — спиннер.
- **Закрытие:** Swipe down с rubber-band эффектом.

## 11. Критерии приёмки

- [ ] Время прибытия (старт − 15 мин) визуально выделено.
- [ ] Мини-карта занимает достаточно места, но не ломает скролл.
- [ ] Sticky CTA «Записаться» всегда видна.
- [ ] Проработаны состояния «Мест нет», «Отменён центром», «Заезд начался».
- [ ] Fallback карты при недоступности сервиса.
- [ ] Анимация раскрытия из карточки списка.

## 12. Артефакты на выходе

1. Компонент «Карточка слота» (Bottom Sheet): все состояния (успех, мест нет, отменён, офлайн).
2. Компонент «Мини-карта места сбора» + fallback.
3. Компонент «Блок времени прибытия» (акцент).
4. Прототип: раскрытие из списка, тап по CTA, закрытие swipe.
5. Спецификация: отступы, цвета, тексты алертов, высота Bottom Sheet.
6. Ассеты: кастомная метка для карты, иконки.
