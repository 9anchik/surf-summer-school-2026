## 9. Технические и платформенные ограничения (NFR)

> *Жёсткие рамки, которые нельзя нарушать.*

- **Тач-зоны:** Минимум **44×44 pt** (iOS) / **48×48 dp** (Android) для всех интерактивных элементов.
- **Safe Area:** Учитывать «чёлку», «островок» и нижнюю зону жестов (Home Indicator).
- **Клавиатуры:** *[Например: Для ввода телефона — цифровая с «+», для OTP — только цифры, авто-переход между полями]*
- **Контрастность:** Текст и иконки должны соответствовать **WCAG 2.1 AA** (критично для чтения на солнце).
- **Системные шрифты:** Поддержка Dynamic Type (iOS) и масштабирования шрифта (Android) без поломки вёрстки.
- **Тёмная тема:** *[Нужна ли? Если да — проработать оба варианта]*

## 10. Анимации и микроинтеракции

> *Как экран «оживает». Не обязательно прорабатывать покадрово, но логику указать.*

- **Переходы между экранами:** *[Например: Карточка слота «раскрывается» в экран деталей (Shared Element Transition)]*
- **Состояния кнопок:** *[Например: При тапе — лёгкое затемнение (ripple/press state), при загрузке — спиннер внутри кнопки]*
- **Обновление данных:** *[Например: Pull-to-refresh с кастомной анимацией (например, вращающееся колесо картинга)]*
- **Появление элементов:** *[Например: Карточки списка появляются с лёгким slide-up с задержкой]*

## 11. Критерии приёмки

> *Чек-лист для самопроверки дизайнера перед передачей в разработку.*

- [ ] Проработаны **все состояния**: загрузка, пустота, ошибка, офлайн, успех.
- [ ] Учтены **альтфернативные потоки** и точки принятия решений из Use Cases.
- [ ] Соблюдены **тач-зоны** (≥44pt) и **контрастность** (WCAG AA).
- [ ] Продуманы **клавиатуры** для всех полей ввода.
- [ ] Учтены **Safe Area** и системные элементы (статус-бар, Home Indicator).
- [ ] Проработаны **Deep Links** из push-уведомлений.
- [ ] Учтены **лимиты и ограничения** (макс. 3 места, нехватка проката, гонка запросов).
- [ ] Дизайн адаптирован под **малые экраны** (iPhone SE / компактные Android).
- [ ] Проработаны **анимации переходов** и состояния кнопок.

## 12. Артефакты на выходе

> *Что именно дизайнер передаёт в разработку.*

1. **UI Kit / Компоненты:** Обновлённые или новые компоненты для дизайн-системы (карточка слота, тоггл экипировки, статус-бейдж).
2. **Макеты экранов:** Все состояния (Light, обязательно; Dark — опционально).
3. **Интерактивный прототип:** (Figma) — для демонстрации Happy Path и ключевых альтернатив.
4. **Спецификация для разработки:**
   - Отступы, размеры, цвета (в переменных).
   - Поведение компонентов (что растягивается, что обрезается, что скрывается).
   - Логика валидации и тексты ошибок (копируемые строки).
   - Описание анимаций (длительность, easing).
5. **Ассеты:** Иконки (SVG), иллюстрации для Empty States (SVG/PNG @2x/@3x).
