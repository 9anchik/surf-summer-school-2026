## 2. Цель экрана и JTBD (Jobs to be Done)

> *Зачем этот экран существует? Какую боль он закрывает?*

- **JTBD пользователя:** Когда я вижу длинный список заездов на неделю, я хочу быстро отфильтровать их по дате, трассе и маршалу, чтобы не скроллить вручную и найти подходящий вариант за пару тапов.
- **Бизнес-цель:** Ускорить путь клиента до выбора слота и бронирования, повысить конверсию из просмотра в запись.
- **Главный вопрос, на который экран отвечает:** «Покажи мне заезды именно на мои условия — нужная дата, нужная трасса, нужный маршал».

---

## 3. Пользователь и контекст использования

> *В каких условиях человек будет смотреть на этот экран? Это критично для UI-решений.*

- **Окружение:** На улице, на ярком солнце, возможно в движении; одна рука занята (сумка, шлем).
- **Эмоциональное состояние:** Сосредоточен, быстро принимает решения.
- **Устройство и хват:** Mobile-first (смартфон в одной руке, большой палец свободен).
- **Время на экране:** 5–15 секунд на выбор параметров и применение.

## 4. Информационная архитектура и контент

> *Какие данные приходят с бэкенда. Дизайнер должен понимать иерархию и объём контента.*

### 4.1. Иерархия информации (от важного к второстепенному)
1. **[Блок 1 — Критично]** Быстрые пресеты дат (Сегодня / Завтра / Выходные / Эта неделя).
2. **[Блок 2 — Важно]** Фильтр по дате (кастомный диапазон «с / по»).
3. **[Блок 3 — Важно]** Конфигурация трассы (короткая / длинная — множественный выбор).
4. **[Блок 4 — Контекст]** Маршал (множественный выбор из справочника).
5. **[Блок 5 — Опционально]** Тоггл «Только свободные места».
6. **[Блок 6 — Действие]** Кнопки «Сбросить» и «Применить».

### 4.2. Поля и данные

| Поле | Обязательность | Формат / Лимиты | Примечание для дизайнера |
| :-- | :-- | :-- | :-- |
| Пресеты дат | Обязательно | Чипы: «Сегодня», «Завтра», «Выходные», «Неделя» | Активный пресет — акцентный цвет |
| Кастомный диапазон | Опционально | Два поля: «с» и «по» (дата) | Открывает стандартный date-picker |
| Конфигурация трассы | Обязательно | Чипы: «Короткая (новички)», «Длинная (опытные)» | Множественный выбор, можно оба |
| Маршал | Обязательно | Список чипов с именами маршалов | Множественный выбор; если справочник пуст — блок скрывается |
| Только свободные | Опционально | Тоггл (по умолчанию выключен) | При включении скрывает слоты с `free_seats = 0` |
| Счётчик применённых фильтров | Обязательно | Цифра в заголовке, например «3 фильтра» | Обновляется при изменении |

### 4.3. Ограничения контента
- Имя маршала — до 20 символов (иначе сокращать с `...`).
- Диапазон дат — не более 30 дней.
- Границы периода дат — включительные, трактуются в локальной зоне центра.

## 5. Пользовательские сценарии (UX Flow)

### 5.1. Основные потоки
> *Идеальный путь пользователя без ошибок.*
1. Пользователь на экране списка заездов (SCR-002) тапает по иконке «Фильтры».
2. Открывается Bottom Sheet с блоками фильтров.
3. Выбирает пресет «Выходные», чип «Длинная трасса», отмечает маршала.
4. Видит обновлённый счётчик «3 фильтра» в заголовке.
5. Тапает «Применить» → Bottom Sheet закрывается, список пересобирается.

### 5.2. Альтернативные потоки
> *Легитимные сценарии, которые отличаются от основного.*
- **A1. Кастомный диапазон дат:** Пользователь тапает по полю «с» или «по» → открывается системный date-picker → после выбора диапазон применяется.
- **A2. Сброс фильтров:** Пользователь тапает «Сбросить» → все фильтры возвращаются к дефолту (без ограничений), список пересобирается, Bottom Sheet остаётся открытым.
- **A3. Закрытие без применения:** Пользователь свайпает вниз или тапает по фону → Bottom Sheet закрывается, фильтры не применяются.
- **A4. Множественный выбор:** Внутри одной группы (конфигурация трассы, маршал) можно выбрать несколько значений — они объединяются по ИЛИ. Между группами — по И.

### 5.3. Точки принятия решений
> *Где пользователь должен сделать выбор, и как мы ему помогаем?*
- **Выбор пресета vs кастомный диапазон:** Пресеты — крупные чипы в один тап, кастомный диапазон — ниже, для точного выбора.
- **Применить vs Сбросить:** Две кнопки внизу — «Сбросить» (вторичная) и «Применить» (акцентная).

## 6. Состояния экрана и компонентов

> *Дизайнер обязан проработать ВСЕ состояния, а не только «идеальную картинку».*

### 6.1. Состояния загрузки (Loading)
- **[Скелетон / Шиммер]:** Пока грузится справочник маршалов — скелетон чипов.
- **[Спиннер]:** Внутри кнопки «Применить» при отправке запроса (кнопка блокируется).

### 6.2. Пустые состояния (Empty States)
- **[Пустой справочник маршалов]:** Блок «Маршал» скрывается полностью.
- **[Нет слотов по фильтрам]:** После применения фильтров список пуст → на SCR-002 показывается empty state «Ничего не найдено. Попробуйте изменить фильтры» с CTA «Сбросить фильтры».

### 6.3. Состояния ошибок (Error States)
- **[Ошибка загрузки справочников]:** Bottom Sheet с текстом «Не удалось загрузить фильтры» и кнопкой «Повторить».
- **[Ошибка применения]:** Тост «Не удалось применить фильтры. Проверьте сеть».

### 6.4. Офлайн-режим (NFR-24)
- **[Баннер]:** Персистентный баннер «Нет подключения».
- **[Доступные действия]:** Можно смотреть ранее загруженные фильтры, но применить их нельзя — кнопка «Применить» неактивна.

### 6.5. Ограничения и лимиты (FR-38)
- Если пользователь выбрал диапазон дат более 30 дней — показать подсказку «Максимальный диапазон — 30 дней» и заблокировать кнопку «Применить».
- Если дата «по» раньше даты «с» — подсветить поле красным.

## 7. Навигация и переходы

> *Как пользователь попадает на этот экран и как уходит с него.*

- **Вход на экран:** Тап по иконке «Фильтры» в правом верхнем углу SCR-002 (списка заездов).
- **Выход с экрана:** Тап по «Применить» / «Сбросить» / Swipe down / Тап по затемнённому фону / Системный Back.
- **Поведение Back:** Возврат к SCR-002 без применения изменений (если не был тап по «Применить»).
- **Deep Links (из push):** Не применимо.

---

## 8. Визуальный язык и настроение (UI / Mood)

> *Общие направления для UI. Дизайнер предлагает решения в рамках этих рамок.*

- **Вайб проекта:** Динамика, уличная культура, быстрый выбор. Чипы — крупные, с тактильным откликом.
- **Цветовая семантика статусов:**
  - **Активный фильтр / Успех:** Яркий акцентный цвет (основной цвет бренда).
  - **Неактивный чип:** Контурный, серый.
  - **Ошибка валидации:** Красная подсветка поля.
  - **Сброшенное состояние:** Серый, нейтральный.
- **Типографика:** Заголовки блоков — средние, полужирные; чипы — читаемые, с достаточным padding.
- **Референсы (Moodboard):** Фильтры в приложениях Avito, Яндекс.Маркет — крупные чипы, множественный выбор, sticky-кнопки внизу.

## 9. Технические и платформенные ограничения (NFR)

> *Жёсткие рамки, которые нельзя нарушать.*

- **Тач-зоны:** Минимум **44x44 pt** (iOS) / **48x48 dp** (Android) для всех чипов и кнопок.
- **Safe Area:** Bottom Sheet не залезает под Home Indicator; кнопки «Применить»/«Сбросить» — в безопасной зоне.
- **Клавиатуры:** Для полей дат — системный date-picker (не кастомный).
- **Контрастность:** Текст и иконки должны соответствовать **WCAG 2.1 AA** (критично для чтения на солнце).
- **Системные шрифты:** Поддержка Dynamic Type (iOS) и масштабирования шрифта (Android) без поломки вёрстки — чипы должны растягиваться по ширине.
- **Тёмная тема:** Опционально, но предпочтительно.

## 10. Анимации и микроинтеракции

> *Как экран «оживает». Не обязательно прорабатывать покадрово, но логику указать.*

- **Переходы между экранами:** Bottom Sheet выезжает снизу с spring-анимацией (высота ~70% экрана).
- **Состояния кнопок:** При тапе — лёгкое затемнение (press state); при загрузке — спиннер внутри кнопки «Применить».
- **Выбор чипа:** Плавная смена цвета (контурный → акцентный) с лёгким scale-эффектом (1.0 → 0.97 → 1.0).
- **Обновление счётчика:** Цифра в заголовке меняется с лёгкой bounce-анимацией.
- **Сброс фильтров:** Все чипы одновременно возвращаются в неактивное состояние (fade).

## 11. Критерии приёмки

> *Чек-лист для самопроверки дизайнера перед передачей в разработку.*

- [ ] Проработаны **все состояния**: загрузка справочников, пустой справочник маршалов, ошибка, офлайн.
- [ ] Учтены **альтернативные потоки**: кастомный диапазон, сброс, закрытие без применения, множественный выбор.
- [ ] Соблюдены **тач-зоны** (>=44pt) и **контрастность** (WCAG AA).
- [ ] Продуманы **date-picker** для полей дат (системный, не кастомный).
- [ ] Учтены **Safe Area** и системные элементы (Home Indicator).
- [ ] Счётчик применённых фильтров обновляется динамически.
- [ ] Учтены **лимиты**: диапазон дат до 30 дней, валидация «по не раньше с».
- [ ] Дизайн адаптирован под **малые экраны** (iPhone SE / компактные Android) — чипы переносятся на новую строку.
- [ ] Проработаны **анимации** появления Bottom Sheet и выбора чипов.

## 12. Артефакты на выходе

> *Что именно дизайнер передаёт в разработку.*

1. **UI Kit / Компоненты:** Чип фильтра (3 состояния: неактивный, активный, disabled), пресет даты, тоггл, кнопка «Применить» / «Сбросить».
2. **Макеты экранов:** Все состояния (загрузка, пустой справочник маршалов, ошибка, офлайн, валидация).
3. **Интерактивный прототип:** (Figma) — для демонстрации Happy Path, множественного выбора, сброса.
4. **Спецификация для разработки:**
   - Отступы, размеры, цвета (в переменных).
   - Поведение чипов (множественный выбор, перенос на новую строку).
   - Логика валидации дат и тексты ошибок (копируемые строки).
   - Описание анимаций (длительность, easing).
5. **Ассеты:** Иконки фильтров, иконка сброса (SVG).
