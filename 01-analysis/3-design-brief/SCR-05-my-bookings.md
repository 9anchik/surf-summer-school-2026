# SCR-05: Мои заезды (Список) — Постановка на дизайн

## 1. Метаданные

| Поле | Значение |
| :-- | :-- |
| **ID экрана** | SCR-05 |
| **Название** | Мои заезды (Список) |
| **Роль пользователя** | Клиент |
| **Тип экрана** | Корневой (Bottom Navigation → Мои заезды) |
| **Связанные US** | US-9, US-11 |
| **Связанные UC** | UC-2 |
| **Ключевые FR** | FR-35a, FR-46 |
| **Связанные NFR** | NFR-24 |

## 2. Цель экрана и JTBD

- **JTBD:** Когда я хочу вспомнить, когда я записан, или посмотреть историю катаний, я хочу увидеть список своих броней с их статусами, чтобы контролировать свои заезды.
- **Бизнес-цель:** Дать клиенту контроль, снизить нагрузку на администратора.
- **Главный вопрос:** «Когда мой следующий заезд? Что у меня было раньше?»

## 3. Пользователь и контекст использования

- **Окружение:** Быстрый чек «что у меня там на выходные?» или «где мой чек за прошлый заезд?».
- **Эмоция:** Контроль, спокойствие.
- **Устройство:** Mobile-first.
- **Время на экране:** 5–15 секунд.

## 4. Информационная архитектура и контент

### 4.1. Иерархия информации
1. **Табы / Сегменты:** Предстоящие | Прошедшие.
2. **Карточка брони:** дата, время, трасса, статус, кол-во мест.

### 4.2. Поля и данные

| Поле | Обязательность | Формат / Лимиты | Примечание |
| :-- | :-- | :-- | :-- |
| Дата и время | Обязательно | `Сб, 15 июня, 18:00` | — |
| Конфигурация трассы | Обязательно | Бейдж | — |
| Статус | Обязательно | Ожидается / Поздняя отмена / Отменён центром / Завершён | Цветовой бейдж |
| Количество мест | Обязательно | `2 места` | — |
| Причина отмены | Опционально | Текстом | Только для «Отменён центром» |

### 4.3. Ограничения контента
- Пагинация: `limit`/`offset` (FR-35a).

## 5. Пользовательские сценарии

### 5.1. Основные потоки
1. Открывается вкладка «Предстоящие».
2. Клиент видит карточку → тапает → SCR-06 (Детали брони).

### 5.2. Альтернативные потоки
- **A1. Переключение табов:** «Предстоящие» ↔ «Прошедшие».
- **A2. Пагинация:** Доскролл до конца → подгрузка следующей порции.
- **A3. Pull-to-refresh:** Обновление списка.

### 5.3. Точки принятия решений
- Выбор карточки → переход в детали.

## 6. Состояния экрана и компонентов

### 6.1. Загрузка
- Скелетон карточек.

### 6.2. Пустые состояния
- **Нет предстоящих:** «У вас пока нет заездов. Самое время выбрать трассу!», CTA «К списку заездов».
- **Нет прошедших:** «Вы ещё не катались с нами. Прошедшие заезды будут здесь».

### 6.3. Ошибки
- **Ошибка загрузки:** «Не удалось загрузить», кнопка «Попробовать снова».

### 6.4. Офлайн-режим
- Баннер «Нет подключения». Показывается кэш.

### 6.5. Ограничения
- **Отменён центром:** Карточка с красным бейджем, видна причина.
- **Поздняя отмена:** Карточка с янтарным бейджем.

## 7. Навигация и переходы

- **Вход на экран:** Тап по иконке «Мои заезды» в Bottom Navigation / Deep link из push (отмена центром).
- **Выход:** Тап по карточке → SCR-06 / Тап по другой иконке Bottom Nav.
- **Поведение Back:** На этом экране системный Back = выход из приложения.
- **Deep Links:** Тап по пушу «Заезд отменён» → открывается этот экран с раскрытой карточкой отменённого заезда.

## 8. Визуальный язык и настроение

- **Вайб:** Контроль, порядок.
- **Цветовая семантика статусов:**
  - 🟢 Ожидается.
  - 🟡 Поздняя отмена.
  - 🔴 Отменён центром.
  - ⚪ Завершён.
- **Типографика:** Дата и время — крупно, статус — бейдж.

## 9. Технические ограничения

- **Тач-зоны:** Карточка — ≥44pt по высоте.
- **Safe Area:** Bottom Navigation в безопасной зоне.
- **Контраст:** WCAG AA.

## 10. Анимации и микроинтеракции

- **Переключение табов:** Плавный slide.
- **Появление карточек:** Каскадный fade-in.
- **Pull-to-refresh:** Кастомный спиннер.

## 11. Критерии приёмки

- [ ] Статусы читаются мгновенно (цвет + текст).
- [ ] Отменённые центром заезды визуально выделяются.
- [ ] Проработаны пустые состояния.
- [ ] Пагинация работает.
- [ ] Deep Link из push отрабатывает.

## 12. Артефакты на выходе

1. Экран списка: все состояния (загрузка, пустота, ошибка, офлайн).
2. Компонент «Карточка брони»: 4 состояния (по статусам).
3. Сегмент-контрол (табы).
4. Прототип: переключение табов, тап по карточке, pull-to-refresh.
5. Спецификация: отступы, цвета статусов, тексты empty states.
