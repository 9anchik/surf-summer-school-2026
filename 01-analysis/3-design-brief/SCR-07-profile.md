# SCR-07: Профиль и Настройки — Постановка на дизайн

## 1. Метаданные

| Поле | Значение |
| :-- | :-- |
| **ID экрана** | SCR-07 |
| **Название** | Профиль и Настройки |
| **Роль пользователя** | Клиент |
| **Тип экрана** | Корневой (Bottom Navigation → Профиль) |
| **Связанные US** | US-17 |
| **Связанные UC** | UC-7 |
| **Ключевые FR** | FR-47, FR-48, FR-49 |
| **Связанные NFR** | NFR-20 |

## 2. Цель экрана и JTBD

- **JTBD:** Когда я хочу изменить номер телефона или удалить аккаунт, я хочу сделать это в пару тапов, чтобы мои данные были актуальны или удалены по моему запросу.
- **Бизнес-цель:** Соответствие требованиям ПДн (право на забвение), актуальность контактов.
- **Главный вопрос:** «Как мне изменить свои данные или удалить аккаунт?»

##  3. Пользователь и контекст использования

- **Окружение:** Спокойная обстановка.
- **Эмоция:** Нейтральная.
- **Устройство:** Mobile-first.
- **Время на экране:** 10–30 секунд.

## 4. Информационная архитектура и контент

### 4.1. Иерархия информации
1. **Аватар / инициалы** (опц.).
2. **Имя и телефон** (редактируемые).
3. **Кнопки действий:** «Редактировать», «Выйти», «Удалить аккаунт».
4. **Ссылка на политику конфиденциальности.**

### 4.2. Поля и данные

| Поле | Обязательность | Формат / Лимиты | Примечание |
| :-- | :-- | :-- | :-- |
| Имя | Обязательно | До 50 символов | Редактируемое |
| Телефон | Обязательно | Маска +7 | Редактируемое (с SMS OTP) |
| Аватар | Опционально | — | Инициалы или фото |

### 4.3. Ограничения контента
- Редактирование телефона требует повторной SMS-верификации.

## 5. Пользовательские сценарии

### 5.1. Основные потоки
1. Клиент открывает профиль.
2. Видит свои данные.
3. Тапает «Редактировать» → inline-редактирование или модалка.
4. Сохраняет → данные обновляются.

### 5.2. Альтернативные потоки
- **A1. Выход из аккаунта:** Тап «Выйти» → модалка подтверждения → возврат на SCR-01.
- **A2. Удаление аккаунта:** Тап «Удалить» → модалка с предупреждением → подтверждение (опц. SMS-код) → удаление → возврат на SCR-01.
- **A3. Изменение телефона:** Тап по телефону → SCR-01 (повторная верификация).

### 5.3. Точки принятия решений
- **Удаление аккаунта** — требует явного подтверждения.

## 6. Состояния экрана и компонентов

### 6.1. Загрузка
- Скелетон профиля.

### 6.2. Пустые состояния
- Не применимо.

### 6.3. Ошибки
- **Ошибка сохранения:** «Не удалось сохранить. Проверьте сеть», кнопка «Повторить».
- **Ошибка удаления:** «Не удалось удалить аккаунт. Попробуйте позже».

### 6.4. Офлайн-режим
- Данные из кэша (read-only). Кнопки действий неактивны.

### 6.5. Ограничения
- **Удаление аккаунта:** Модалка с предупреждением «Все активные брони будут отменены, история анонимизирована. Вы уверены?».

## 7. Навигация и переходы

- **Вход на экран:** Тап по иконке «Профиль» в Bottom Navigation.
- **Выход:** Тап по другой иконке Bottom Nav.
- **Поведение Back:** На этом экране системный Back = выход из приложения.
- **После выхода/удаления:** Переход на SCR-01.

## 8. Визуальный язык и настроение

- **Вайб:** Спокойствие, контроль, прозрачность.
- **Цветовая семантика:**
  - 🔴 Удаление аккаунта — красный (опасное действие).
  - ⚪ Выход — серый.
- **Типографика:** Имя — H2, телефон — body.

## 9. Технические ограничения

- **Тач-зоны:** Кнопки — ≥44pt.
- **Клавиатуры:** Для имени — стандартная, для телефона — цифровая с маской.
- **Контраст:** WCAG AA.

## 10. Анимации и микроинтеракции

- **Редактирование:** Плавное появление полей ввода.
- **Модалка удаления:** Slide-up.
- **CTA:** Press state.

## 11. Критерии приёмки

- [ ] Проработаны сценарии редактирования, выхода, удаления.
- [ ] Модалка удаления с предупреждением.
- [ ] Защита от случайного удаления (опц. SMS-код).
- [ ] Ссылка на политику конфиденциальности.

## 12. Артефакты на выходе

1. Экран профиля: все состояния (просмотр, редактирование, офлайн).
2. Модалки: выход, удаление аккаунта.
3. Прототип: редактирование, выход, удаление.
4. Спецификация: отступы, тексты модалок.
