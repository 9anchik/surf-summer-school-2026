# SCR-06: Детали брони и Отмена — Постановка на дизайн

## 1. Метаданные

| Поле | Значение |
| :-- | :-- |
| **ID экрана** | SCR-06 |
| **Название** | Детали брони и Отмена |
| **Роль пользователя** | Клиент |
| **Тип экрана** | Детализация / Действие |
| **Связанные US** | US-10, US-11 |
| **Связанные UC** | UC-2 |
| **Ключевые FR** | FR-16, FR-17, FR-18, FR-46 |
| **Связанные NFR** | NFR-21 |

## 2. Цель экрана и JTBD

- **JTBD:** Когда мои планы изменились, я хочу быстро отменить заезд (или понять, что будет при поздней отмене), чтобы не потерять деньги и не подвести центр.
- **Бизнес-цель:** Честно показать политику отмен (3 часа), освободить места для других.
- **Главный вопрос:** «Могу ли я отменить? Что будет, если отменю сейчас?»

## 3. Пользователь и контекст использования

- **Окружение:** Возможно, стресс (планы изменились).
- **Эмоция:** Срочность, ответственность.
- **Устройство:** Mobile-first.
- **Время на экране:** 10–30 секунд.

## 4. Информационная архитектура и контент

### 4.1. Иерархия информации
1. **Детали брони:** слот, экипировка, цена.
2. **Таймер до старта:** «До заезда осталось 3 ч 15 мин».
3. **Политика отмены:** Динамический текст.
4. **CTA:** «Отменить заезд».

### 4.2. Поля и данные

| Поле | Обязательность | Формат / Лимиты | Примечание |
| :-- | :-- | :-- | :-- |
| Дата и время | Обязательно | `Сб, 15 июня, 18:00` | — |
| Конфигурация трассы | Обязательно | Бейдж | — |
| Маршал | Обязательно | Имя | — |
| Количество мест | Обязательно | `2 места` | — |
| Экипировка | Обязательно | `1 своя, 1 прокат` | — |
| Итоговая цена | Обязательно | `5 600 ₽` | — |
| Таймер до старта | Обязательно | `3 ч 15 мин` | Обновляется |
| Политика отмены | Обязательно | Динамический текст | Зависит от таймера |
| Причина отмены | Опционально | Текстом | Только для «Отменён центром» |

### 4.3. Ограничения контента
- Таймер обновляется в реальном времени.
- Граница 3 часа — ровно включительно (FR-17).

## 5. Пользовательские сценарии

### 5.1. Основные потоки
1. Клиент открывает детали брони.
2. Видит таймер и политику отмены.
3. Тапает «Отменить заезд» → модалка подтверждения.
4. Подтверждает → успех → возврат в SCR-05.

### 5.2. Альтернативные потоки
- **A1. Ранняя отмена (≥3 ч):** Модалка «Отменить заезд? Место освободится».
- **A2. Поздняя отмена (<3 ч):** Модалка с предупреждением «Вы отменяете менее чем за 3 часа. Карт будет простаивать. Продолжить?».
- **A3. Отмена не нужна:** Клиент закрывает экран.

### 5.3. Точки принятия решений
- **Полистика отмены** — должна быть понятна до тапа по CTA.
- **Модалка подтверждения** — защита от случайного тапа.

## 6. Состояния экрана и компонентов

### 6.1. Загрузка
- Скелетон экрана.

### 6.2. Пустые состояния
- Не применимо.

### 6.3. Ошибки
- **Ошибка отмены:** Bottom Sheet «Не удалось отменить. Проверьте сеть», кнопка «Повторить».
- **Бронь уже отменена:** Автоматический возврат в SCR-05 с алертом.

### 6.4. Офлайн-режим
- CTA «Отменить» неактивна, баннер «Нет подключения».

### 6.5. Ограничения
- **Заезд уже начался:** CTA скрыта, статус «Заезд начался».
- **Отменён центром:** CTA скрыта, показана причина.
- **Уже отменена:** CTA скрыта, статус «Отменена».

## 7. Навигация и переходы

- **Вход на экран:** Тап по карточке в SCR-05 / Deep link из push (напоминание).
- **Выход:** Системный Back / Swipe back → SCR-05.
- **Поведение Back:** Возврат в список броней.
- **Deep Links:** Тап по пушу «Через 3 часа заезд» → этот экран с конкретной бронью.

## 8. Визуальный язык и настроение

- **Вайб:** Забота, но строгость. Прозрачность правил.
- **Цветовая семантика:**
  - 🟢 Ранняя отмена (свободно).
  - 🟡 Поздняя отмена (внимание).
  - 🔴 Отменён центром / заезд начался.
- **Типографика:** Таймер — крупно, моноширинный.

## 9. Технические ограничения

- **Тач-зоны:** CTA — ≥44pt.
- **Safe Area:** CTA не перекрывается Home Indicator.
- **Контраст:** WCAG AA.

## 10. Анимации и микроинтеракции

- **Таймер:** Плавное обновление (каждую минуту).
- **Модалка подтверждения:** Slide-up.
- **CTA:** Press state, при загрузке — спиннер.

## 11. Критерии приёмки

- [ ] Динамическое изменение текста политики отмены.
- [ ] Модалка подтверждения с разным текстом для ранней/поздней отмены.
- [ ] Проработаны состояния «Заезд начался», «Отменён центром», «Уже отменена».
- [ ] Deep Link из push отрабатывает.

## 12. Артефакты на выходе

1. Экран деталей: все состояния (ранняя/поздняя отмена, отменён центром, заезд начался).
2. Модалка подтверждения: 2 варианта (ранняя/поздняя).
3. Компонент «Таймер до старта».
4. Прототип: тап по CTA, модалка, успех.
5. Спецификация: отступы, тексты модалок, логика таймера.
