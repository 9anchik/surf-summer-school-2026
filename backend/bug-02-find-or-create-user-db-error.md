# Bug 02: Ошибка БД при поиске пользователя приводит к попытке создать нового пользователя

## Симптом

В `FindOrCreateUser` после `SELECT users WHERE phone = $1` любая ошибка воспринимается как отсутствие пользователя. После этого код выполняет `INSERT`.

## Почему это критично

Если чтение пользователя упало из-за временной ошибки БД, проблемы соединения или отмены контекста, сервис может ошибочно попытаться создать пользователя заново. В лучшем случае это приведёт к конфликту `UNIQUE(phone)`, в худшем — к неконсистентному поведению авторизации и невозможности корректно войти существующему пользователю.

## Цель исправления

Продолжать создание пользователя только если ошибка равна `pgx.ErrNoRows`. Любая другая ошибка должна возвращаться из метода без попытки `INSERT`.
